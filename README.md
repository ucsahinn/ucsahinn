## Çalışma Standardı

<table>
  <tr>
    <td width="33%">
      <h3>Security first</h3>
      <p>Secrets stay private, source-sensitive products publish only public documentation, release artifacts and support surfaces.</p>
    </td>
    <td width="33%">
      <h3>Release discipline</h3>
      <p>Installer, manifest, hash, docs, support policy and remote state are treated as one release surface.</p>
    </td>
    <td width="33%">
      <h3>Readable operations</h3>
      <p>Tools should be understandable by operators, not only by the person who built them.</p>
    </td>
  </tr>
</table>
